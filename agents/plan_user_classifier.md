# Variables
- $$idea = 사용자의 러프한 아이디어 텍스트
- $$requirements = plan_requirement_analyzer의 결과 (서비스 개요 + FR/NFR 목록)
- $$depth = 기획 깊이 (light / standard / deep)

# Rules
- $$variable 형식으로 변수 참조
- 각 Step 완료후 다음 Step 진행 전 결과를 명시적으로 서술.
- $$depth에 따라 산출물의 상세 수준을 조절한다.
  - light: 주요 사용자 유형 2~3개, 페르소나 간략 정의
  - standard: 사용자 유형 3~5개, 페르소나 상세 정의
  - deep: 사용자 유형 5개 이상, 페르소나 심층 정의 + 근거 및 참고자료 포함

## Errors/Exception Handling
- $$requirements가 부족하여 사용자 유형 추론 불가 → 부모 Context에 보고, 보완 요청
- 도메인 리서치 실패 → 리서치 없이 $$requirements 기반으로 진행, 보고

# Action

## Step 1. 서비스 컨텍스트 분석
$$idea와 $$requirements를 분석하여 아래 항목을 파악한다:
- **서비스 도메인**: $$requirements의 서비스 개요에서 추출
- **핵심 기능 목록**: FR 중 Core 카테고리 기능 요약
- **서비스가 해결하는 문제**: 핵심 가치 제안 재확인
- **잠재 사용자 풀**: 이 서비스를 사용할 수 있는 모든 이해관계자 나열

## Step 2. 사용자 유형 도출
잠재 사용자 풀을 기반으로 사용자 유형을 분류한다.

### 분류 기준
- **역할(Role)**: 서비스 내에서의 역할 (예: 소비자, 제공자, 관리자)
- **목적(Goal)**: 서비스를 사용하는 주된 목적
- **빈도(Frequency)**: 사용 빈도 (일상적 / 간헐적 / 일회성)
- **숙련도(Proficiency)**: 디지털 리터러시 수준 (초보 / 중급 / 고급)

### 출력 형식
```
[UT-{번호}] {사용자 유형명}
- 역할: {서비스 내 역할}
- 목적: {서비스 사용 주 목적}
- 빈도: 일상적 / 간헐적 / 일회성
- 숙련도: 초보 / 중급 / 고급
- 비중: {전체 사용자 중 예상 비율}
```

## Step 3. 페르소나 정의
각 사용자 유형에 대해 구체적인 페르소나를 정의한다.

### 페르소나 항목
```
[P-{번호}] {페르소나 이름} ({사용자 유형 UT-{번호} 대표})
- 나이/성별: {예시 인물상}
- 직업/배경: {직업 또는 생활 배경}
- 핵심 니즈: {이 서비스에서 가장 원하는 것}
- 불편 사항(Pain Point): {현재 겪고 있는 문제점}
- 기대 효과(Gain): {서비스를 통해 얻고자 하는 가치}
- 사용 시나리오: {서비스를 언제, 어디서, 어떻게 사용하는지 1~2문장}
- 기술 환경: {주로 사용하는 디바이스, 브라우저, OS 등}
```

> $$depth가 deep인 경우, 각 페르소나에 대해 아래 항목을 추가:
> - 행동 동기(Motivation): 심리적/사회적 동기
> - 이탈 요인(Churn Risk): 서비스 이탈 가능성이 높은 상황
> - 참고 사례: 유사 서비스에서의 사용자 행동 패턴 (URL 포함)

## Step 4. 사용자 유형 간 관계 매핑
사용자 유형 간의 상호작용 관계를 정리한다:
- **직접 상호작용**: 서비스 내에서 직접 소통/거래하는 관계
- **간접 의존**: 한 유형의 활동이 다른 유형에 영향을 미치는 관계
- **관계도**: 유형 간 관계를 텍스트 다이어그램으로 표현

```
예시)
[소비자] ←주문/결제→ [제공자]
[제공자] ←등록/관리→ [관리자]
[관리자] ←정책/알림→ [소비자]
```

## Step 5. 사용자 유형 요약 및 검증
도출된 결과를 종합 정리한다:
- 사용자 유형 총 수
- 페르소나 총 수
- 주요 사용자 유형 (비중 상위)
- 누락 가능성이 있는 사용자 유형 명시

## Step 6. 부모 Context로 전달
아래 구조로 결과를 부모 Context에 반환한다:
```
## 사용자 유형 분류 결과

### 사용자 유형 목록
[UT-001] ...
[UT-002] ...
...

### 페르소나 정의
[P-001] ...
[P-002] ...
...

### 사용자 유형 간 관계
(관계도)

### 요약
- 사용자 유형: N개
- 페르소나: N개
- 주요 사용자: UT-{번호} ({비중}%)
```
