# 피드백 루프 프로토콜

Skill의 마지막 단계 또는 피드백 Step에서 아래 4단계 루프를 적용한다.

## N-1. 피드백 수집
- Human에게 산출물에 대한 항목별 피드백을 요청한다:
  - 산출물 내용이 의도에 부합하는가
  - 수정/보완이 필요한 부분이 있는가
  - 삭제하거나 변경할 항목이 있는가

## N-2. 영향 범위 분석
- 피드백 내용을 분석하여 수정 범위를 판정한다:
  - 내용 보완 → 해당 파일/섹션만 수정
  - 구조 변경 → 영향받는 파일 전체 재생성
  - 전체 방향 변경 → 관련 Step부터 재실행
- 수정 계획을 Human에게 제시하고 확인받는다.

## N-3. 선택적 재실행
- 확인된 수정 계획에 따라 해당 부분만 수정/재생성한다.
- 수정 시 이전 내용과의 변경점(diff)을 명시한다.
- 재실행 결과를 후속 단계에 전파하여 일관성을 유지한다.

## N-4. 반복 판정
- 수정 결과를 Human에게 제시한다.
- Human이 승인하면 최종 결과를 확정하고 종료한다.
- 추가 피드백이 있으면 N-1로 복귀하여 루프를 반복한다.
- 최대 반복 횟수: SKILL에서 별도 지정하지 않으면 3회 (초과 시 Human에게 알리고 현재 결과로 확정)
