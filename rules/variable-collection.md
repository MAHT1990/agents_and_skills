# Step 0 요구사항 회의 프로토콜

Variables 섹션에 정의된 변수를 수집하고, Human과의 회의를 통해 요구사항을 구체화하는 Step 0에 공통 적용한다.

## 0-1. 변수 수집
- SKILL.md의 Variables 섹션에 "(필수)"로 표기된 변수는 반드시 확보한다.
- 필수 변수가 모두 확보될 때까지 회의 단계로 진행하지 않는다.
- default 값이 명시된 변수는 Human이 별도 지정하지 않으면 default 값을 적용한다.
- "(선택)" 또는 "(optional)"로 표기된 변수는 미제공 시 스킵 가능하다.
- Human이 일부 변수를 프롬프트에 이미 포함한 경우, 해당 값을 자동 추출하여 재질문하지 않는다.

## 0-2. 요구사항 구체화 회의 (루프)
수집된 변수를 바탕으로 Human과 요구사항을 구체화하는 회의를 진행한다.
이 회의는 Human이 최종 승인할 때까지 반복한다.

### 회의 진행 규칙
1. 아래 항목을 Human에게 질의하여 요구사항을 구체화한다:
   - 작업 목적과 기대 결과
   - 모호하거나 누락된 요구사항
   - 제약 조건이나 특이사항
   - SKILL.md에 스킬 고유의 회의 항목이 정의되어 있으면 해당 항목도 포함
2. Human의 답변을 반영하여 요구사항을 갱신한다.
3. 갱신된 요구사항에 대해 추가 질의가 필요하면 1로 돌아간다.
4. 더 이상 질의 사항이 없으면 0-3(최종 승인)으로 진행한다.

## 0-3. 최종 승인
확정된 요구사항을 아래 형식으로 Human에게 제시하고 **최종 승인**을 받는다:
```
━━━ Step 0 요구사항 확인서 ━━━
• 변수 목록: {수집된 변수와 확정 값}
• 작업 목적: {회의에서 확인된 목적}
• 기대 결과: {회의에서 확인된 기대 산출물}
• 제약/특이사항: {회의에서 확인된 제약 조건}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
위 내용으로 진행해도 되겠습니까? (승인/수정)
```
- **승인 없이 다음 Step으로 진행하지 않는다.**
- "수정" 시, 수정 사항을 반영하고 0-2(회의)로 돌아가 재논의 후 다시 승인을 요청한다.
