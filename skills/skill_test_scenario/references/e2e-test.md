# E2E Test Scenario 작성 가이드

## 목적
실제 사용자 관점에서 애플리케이션의 전체 흐름을 검증하는 테스트 시나리오를 도출한다.
브라우저/클라이언트에서 시작하여 서버, DB까지 전 구간을 대상으로 한다.

## 분석 기준
### Step1. 페이지 구조 분석
PlayWright MCP를 이용하여 각 URI 페이지를 분석:
- 주요 UI 컴포넌트 (폼, 버튼, 테이블, 모달 등)
- 사용자 행동 흐름 (입력 → 버튼 클릭 → 결과)
- 인증/권한 분기 (로그인 필요 여부, 관리자/일반 사용자)
- API 호출 지점

### Step2. 시나리오 유형
| 유형 | 설명 | 예시 |
|---|---|---|
| Happy Path | 정상 흐름 전체 커버 | 로그인 → 메뉴 진입 → 데이터 조회 |
| Negative | 잘못된 입력/접근 | 필수값 누락, 권한 없는 페이지 접근 |
| Edge Case | 경계값, 특수 상태 | 빈 데이터, 대량 데이터, 네트워크 지연 |
| Cross-Page | 페이지 간 연동 | A 페이지에서 생성 → B 페이지에서 확인 |

### Step3. TC 형식
```
[TC-{번호}] {시나리오 제목}
- 유형: Happy Path / Negative Case / Edge Case / Cross-Page
- 사전 조건: (예: 로그인 필요, 특정 데이터 존재 등)
- 입력값: ...
- 테스트 절차: ...
- 예상 결과: ...
```

## 주의사항
- 테스트 환경(staging/production) 명시
- 테스트 데이터 사전 준비 사항 명시
- 민감 정보(ID/PW 등) 절대 코드/파일에 저장 금지
- 페이지 로딩 시간, 비동기 처리 대기 고려
